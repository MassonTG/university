Роль кожного шару
Presentation (UI)

Це інтерфейс для користувачів та адміністраторів.

Користувачі можуть переглядати книги, робити бронювання, а адміністратори — додавати чи редагувати книги.

UI не містить бізнес‑логіки, а лише відображає дані та надсилає запити до сервісів.

Business Logic (сервіси, правила)

Тут реалізуються правила системи: бронювання книг, перевірка доступності, управління користувачами.

Сервіси координують роботу між UI та Data Access.

Наприклад, ReservationService перевіряє, чи книга доступна, і створює бронювання.

Data Access (робота з даними)

Відповідає за взаємодію з базою даних.

Repository класи зберігають та отримують інформацію про книги, користувачів і бронювання.

Цей шар ізольований від бізнес‑логіки, що дозволяє легко змінювати базу даних без впливу на інші частини системи.

Взаємодія між шарами
Користувач через UI надсилає запит (наприклад, «забронювати книгу»).

Business Logic (ReservationService) перевіряє правила (чи книга доступна, чи користувач має право).

Якщо все ок, сервіс звертається до Data Access (ReservationRepository) для збереження бронювання.

Результат повертається назад у UI, де користувач бачить підтвердження.
