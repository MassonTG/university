MVP To-Do List (Windows Forms)

Опис  
Цей проект демонструє архітектурний шаблон MVP (Model–View–Presenter) на прикладі простого списку завдань. Користувач може додавати завдання, переглядати їх у списку та відмічати виконані.

Принцип MVP  
Model (TaskModel) зберігає дані про завдання.  
View (Form1 та Designer) відповідає за інтерфейс користувача і лише показує дані.  
Presenter (TaskPresenter) містить бізнес-логіку, працює з моделлю і передає дані у View.  
Таким чином модель не знає про інтерфейс, View не містить логіки, а Presenter керує всією взаємодією.

Як запустити  
Відкрий Visual Studio і створіть новий проект Windows Forms App (.NET Framework). Назву проекту задай MVP_ToDoList. Після створення у рішенні буде файл Form1.cs і Program.cs. . Ми замінимо їх кодом і додамо нові файли.

Створи новий файл TaskModel.cs у корені проекту. Встав код моделі
Створи новий файл ITaskView.cs у корені проекту. Встав інтерфейс представлення
Створи новий файл TaskPresenter.cs у корені проекту. Встав код презентера
Відкрий існуючий файл Form1.cs і заміни його вміст на логіку форми, щоб вона реалізовувала ITaskView і викликала презентер
Створи або відредагуй файл Form1.Designer.cs. . Якщо він уже існує, відкрий і заміни метод InitializeComponent на цей, щоб були потрібні елементи інтерфейсу і події
Переконайся, що файл Form1.cs має partial клас Form1 і методи btnAdd_Click та listBoxTasks_DoubleClick збігаються з підписами у Designer. Якщо Visual Studio згенерує власний Designer, просто заміни властивості і підпишись на події як показано.
Відкрий Program.cs і заміни вміст на стандартну точку входу, яка запускає форму
Переконайся, що у всіх файлах однаковий namespace MVP_ToDoList. Якщо проект має іншу назву простору імен, або ти розмістив файли у папках, вирівняй namespace однаково або онови using.
Збережи всі файли. Натисни Start або F5 для запуску. У відкритій формі введи текст задачі у поле, натисни кнопку Add. Записи з’являються у списку. Подвійний клік по запису перемикає статус виконано чи не виконано.
