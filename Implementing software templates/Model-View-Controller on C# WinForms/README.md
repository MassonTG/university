MVC Список задач у Windows Forms додптку

 Опис
Цей проект демонструє архітектурний шаблон **MVC (Model–View–Controller)** на прикладі простого списку завдань.  
Користувач може додавати завдання, переглядати їх у списку та відмічати виконані.

 Принцип MVC
- Model (TaskModel): зберігає дані про завдання (опис, статус).  
- View (Form1 + Designer): відповідає за інтерфейс користувача (текстове поле, кнопка, список).  
- Controller (TaskController): керує логікою — додає завдання, змінює статус, оновлює View.  

Таким чином, ми розділяємо відповідальності:  
Модель не знає про інтерфейс.  
View не містить бізнес-логіки.  
Controller виступає посередником між ними.

Як запустити
1. Відкрити проект у Visual Studio.  
2. Запустити програму (Form1).  
3. У вікні програми можна:
   - Ввести текст завдання і натиснути Add.  
   - Подвійним кліком по завданню змінити його статус (виконано/не виконано).
