MVP To-Do List (Windows Forms)
Опис
Цей проект демонструє архітектурний шаблон **MVP (Model–View–Presenter)** на прикладі простого списку завдань.  
Користувач може додавати завдання, переглядати їх у списку та відмічати виконані.

Принцип MVP
- Model (TaskModel): зберігає дані про завдання (опис, статус).  
- View (Form1 + Designer): відповідає за інтерфейс користувача (текстове поле, кнопка, список). View є "пасивним" — воно лише показує дані, які отримує від Presenter.  
- Presenter (TaskController): містить бізнес-логіку — додає завдання, змінює статус, оновлює View. Presenter отримує дані з моделі, готує їх і передає у View.  

Таким чином, ми розділяємо відповідальності:  
- Модель не знає про інтерфейс.  
- View не містить бізнес-логіки, лише відображає дані.  
- Presenter виступає посередником між ними, керуючи всією взаємодією.  

Як запустити
1. Відкрити проект у **Visual Studio**.  
2. Запустити програму (Form1).  
3. У вікні програми можна:
   - Ввести текст завдання і натиснути **Add**.  
   - Подвійним кліком по завданню змінити його статус (виконано/не виконано).
