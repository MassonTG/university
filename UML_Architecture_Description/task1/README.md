Опис предметної області
Система управління бібліотекою дозволяє користувачам переглядати доступні книги, брати їх у користування та повертати. 
Адміністратор може додавати нові книги, оновлювати інформацію про них та контролювати терміни користування. 
Кожна книга має унікальний ідентифікатор, автора та статус (доступна чи позичена). 
Бронювання книги створюється користувачем і пов’язане з датою видачі та повернення.

Архітектурний підхід
Така система може бути реалізована як клієнт-серверна з шаровою архітектурою:

UI (клієнт) — веб‑інтерфейс або мобільний додаток для користувачів і адміністраторів.

Бізнес‑логіка — обробка бронювань, контроль термінів повернення, управління статусами книг.

Дані — база даних із записами про книги, користувачів та бронювання.
